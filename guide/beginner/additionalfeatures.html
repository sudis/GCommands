# Additional features

Accessories that may be useful to you.

# Slash return

For example, do you want when you send a command to be sent as a clyde or to disable ping in a certain message?

module.exports = {
    name: "ping",
    description: "command ping",
    cooldown: 5,
    run: async(client, slash, message, args) => {
      return {
        content: "pong", //or embed object new Discord.MessagEmbed().setTitle("a")
        ephemeral: true, //clyde
        allowedMentions: { parse: [], repliedUser: true }
      }
  }
};

# Cooldowns

Spam is something you generally want to avoid–especially if one of your commands requires calls to other APIs or takes a bit of time to build/send. Cooldowns are also a very common feature bot developers want to integrate into their projects, so let's get started on that!

Add a cooldown key to one of your commands. This determines how long the user will have to wait (in seconds) before using this specific command again.




 





module.exports = {
    name: "ping",
    description: "command ping",
    cooldown: 5,
    run: async(client, slash, message, args) => {
      // ...
  }
};

# Categories

You can have all the commands in the cmdDir folder that you specified. If you want more order, just create another folder in this folder and put commands in it.

# Aliases

If you want the command to be able to be executed from multiple commands. Just add the aliases parameter.




 





module.exports = {
    name: "ban",
    description: "user ban",
    aliases: ["banuser","userban"],
    run: async(client, slash, message, args) => {
        // ...
  }
};

# UserPermissions

Add a requiredPermission key to your existing command options. We will use the 'ban' command to example.




 





module.exports = {
    name: "ban",
    description: "user ban",
    requiredPermission: "ADMINISTRATOR",
    run: async(client, slash, message, args) => {
        // ...
  }
};

Add a userRequiredPermissions key to your existing command options. We will use the 'ban' command to example.




 





module.exports = {
    name: "ban",
    description: "user ban",
    userRequiredPermissions: "ADMINISTRATOR", //["ADMINISTRATOR","MANAGE_GUILD"]
    run: async(client, slash, message, args) => {
        // ...
  }
};

# GuildOnly

If you only want the command for a guild, just add the guildOnly parameter.




 





module.exports = {
    name: "refresh",
    description: "refresh",
    guildOnly: "guild id",
    run: async(client, slash, message, args) => {
        // ...
  }
};

# UserOnly

If you only want the command for a user, just add the userOnly parameter.




 





module.exports = {
    name: "eval",
    description: "run code",
    userOnly: "user id", //["user id2", "user id2"]
    run: async(client, slash, message, args) => {
        // ...
  }
};

# Custom Language File

Do you want to customize the messages for you? So just do the following things.

new GCommands(client, {
    language: "czech", //english, spanish, portuguese, russian, german, czech
    ownLanguageFile: require("./message.json")
})
